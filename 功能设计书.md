这个想法非常实用，语言学习类工具在 LINE 的生态（LIFF - LINE Front-end Framework 或 LINE Mini App）中有着天然的留存优势，因为用户可以随时在聊天间隙打开复习。

在开始详细设计之前，需要先**纠正一个技术前提**：由于 iOS 和 Android 的系统级隐私沙盒限制，LINE 小程序（本质是内嵌的 Web 应用）**无法直接读取或识别用户的“当前屏幕”**。
**替代方案**：用户可以截屏，然后通过“上传图片”功能选中该截图；或者你可以开发一个 LINE Bot，用户直接把截图发送给这个 Bot，Bot 处理后返回带有假名标注的页面链接。

下面是为你梳理的项目设计书，侧重于功能、外观和交互逻辑。

---

## 1. 核心功能设计 (Functional Design)

我们将项目拆解为三个核心模块：**采集与识别**、**注音与解析**、**收藏与复习**。

### 1.1 图像采集与文本提取 (OCR)

* **拍照获取**：调用系统相机实时拍摄纸质书、菜单或标志。
* **相册上传**：支持用户上传已有的图片或手机截图（替代直接屏幕识别）。
* **图像裁剪**：上传后允许用户框选特定区域，避免不相关的背景文字干扰 OCR（光学字符识别）的准确度。

### 1.2 文本注音与展示

* **分词与注音**：对提取出的日文句子进行精准分词（Tokenization）。为所有“汉字”自动生成上标假名（Furigana）和罗马字（Romaji）。
* **动态排版**：支持多行长文本的滑动阅读。罗马字可以选择全局开启或隐藏，避免界面过度拥挤。

### 1.3 交互式收藏与生词本

* **点选查词**：将整段文本打散成可点击的“词汇块”。
* **一键入库**：点击特定词汇后，弹出该词的完整信息（汉字、假名、罗马音、基础释义），并提供“加入收藏”按钮。
* **复习模式**：在独立的“生词本”页面，按时间或记忆曲线（如简单、困难）对收藏的词汇进行列表展示和卡片翻转复习。

## 2. 界面与外观设计 (UI Design)

整体风格建议遵循 LINE 的设计规范（Line Design System），采用干净、轻量的视觉语言，背景以灰白为主，主色调可以使用 LINE 的品牌绿（#06C755）或柔和的护眼色。

| 页面名称 | 核心视觉元素 | 布局说明 |
| --- | --- | --- |
| **首页 (Home)** | 大尺寸功能入口按钮 | 顶部放置“拍照”与“相册”两个大卡片；下方显示“最近识别的文本”历史记录列表。 |
| **识别结果页** | 核心阅读区 + 底部操作栏 | 占据屏幕 80% 的滚动阅读区（大字号，带有假名上标和罗马字下标）；底部悬浮两个按钮：“复制全文”和“隐藏罗马音”。 |
| **词汇详情抽屉** | 半屏弹窗 (Bottom Sheet) | 用户点击某个汉字词语时从底部滑出。显示：词汇、发音、发音喇叭图标（TTS朗读）、大号“⭐ 收藏”按钮。 |
| **生词本页** | 列表与过滤标签 | 顶部提供“全部/按日期/按五十音”切换标签；主体为生词列表；支持左滑删除或取消收藏。 |

## 3. 核心交互逻辑 (UX Flow)

这里是用户从打开小程序到完成一次“识别-学习-收藏”的完整体验路径：

1. **入口与采集:** 首页操作.
用户打开 LINE 小程序，点击“相册上传”，选择了一张刚才在看日剧时截的带有日文字幕的屏幕截图。


2. **云端处理:** OCR与NLP.
界面显示轻量级的加载动画（如扫描线扫过图片）。后台快速调用 OCR 接口提取文本，并将其送入日文形态素分析引擎，将所有汉字与对应的假名及罗马音绑定。


3. **沉浸式阅读:** 结果展示.
页面跳转至“结果页”。截图上的字幕变成了干净的排版文本，汉字上方清晰地标注了假名，下方标注了罗马字。


4. **交互与收藏:** 词汇捕获.
用户遇到不认识的词，手指点击该词汇。该词汇高亮变色，屏幕底部滑出抽屉，显示该词的发音和释义。用户点击抽屉里的“⭐ 收藏”按钮，抽屉收起，回到阅读流。

## 4. 技术实现思路 (简述)

为了支撑上述功能，你需要以下技术组件：

* **前端**：LINE LIFF 框架 + React 或 Vue，利用 `ruby` 和 `rt` HTML 标签来原生实现漂亮的假名上标排版。
* **OCR 引擎**：推荐 Google Cloud Vision API，对带有复杂背景的日文图片识别率极高。
* **注音与分词**：需要一个日文形态素分析器，如 `Kuromoji`（Node.js 可用）或 Yahoo Japan 的 Furigana API，用来把一整句话准确切分并标注读音。
* **后端与存储**：Node.js + Firebase 或 PostgreSQL，用于存储用户的 LINE ID 和他们收藏的生词列表。
