# LINE 小程序 (LINE Mini App) 开发到发布全流程指南

本文档提供从零开始构建、开发、测试到最终通过审核并发布 LINE 小程序（LINE Mini App）的标准操作流程 (SOP)。

---

## 阶段一：前期准备与账号注册

1. **注册 LINE 开发者账号**
   * 访问 [LINE Developers Console](https://developers.line.biz/console/)。
   * 使用你的 LINE 账号登录，并完成开发者注册。
2. **创建 Provider (服务提供商)**
   * 在控制台首页，点击 `Create a new provider`。
   * 输入你的公司名称、团队名称或个人开发者名称。
3. **关联 LINE 官方账号 (Official Account, OA)**
   * LINE 小程序通常需要与一个 LINE 官方账号绑定（用于发送服务消息或积累粉丝）。
   * 在 [LINE Official Account Manager](https://manager.line.biz/) 中创建一个免费的官方账号。

---

## 阶段二：创建 LINE Mini App 渠道 (Channel)

1. **新建 Channel**
   * 在刚才创建的 Provider 下，选择 `Create a new channel`。
   * 选择 **LINE Mini App**（如果你的区域/权限不可见，可先选择 **LINE Front-end Framework (LIFF)**，核心开发逻辑一致，后续可申请升级为 Mini App）。
2. **填写基础信息**
   * 填写 App 名称、描述、分类。
   * 上传 App Icon。
   * 填写隐私政策 (Privacy Policy) 和 服务条款 (Terms of Use) 的 URL（提审必备）。
3. **获取核心凭证**
   * 创建成功后，你会获得一个 **LIFF ID**（格式如 `1234567890-Abcdefgh`），这是前端 SDK 初始化的关键。

---

## 阶段三：环境搭建与代码开发

1. **前端集成 LIFF SDK**
   * 在你的前端项目 (React/Vue/纯 HTML) 中引入 LIFF SDK：
     ```bash
     npm install @line/liff
     ```
   * 初始化 SDK：
     ```javascript
     import liff from '@line/liff';

     async function initializeLiff() {
       try {
         await liff.init({ liffId: "你的_LIFF_ID" });
         if (!liff.isLoggedIn()) {
           liff.login(); // 引导用户授权登录
         } else {
           const profile = await liff.getProfile();
           console.log("用户信息:", profile);
         }
       } catch (error) {
         console.error("LIFF 初始化失败", error);
       }
     }
     

```

2. **配置 Endpoint URL**
* LINE 小程序本质是运行在 LINE App 内置浏览器中的 Web 应用。
* 在开发阶段，使用内网穿透工具（如 `ngrok` 或 `Cloudflare Tunnels`）将本地 `localhost:3000` 映射为一个 **HTTPS** 域名。
* 回到 LINE 开发者控制台，在 Channel 的 `LIFF` 标签页下，将你的 HTTPS 链接填入 `Endpoint URL`。



---

## 阶段四：本地真机调试

1. **添加测试成员**
* 在控制台的 `Roles` 标签页，将你的测试人员的 LINE 账号添加为 `Tester` 或 `Admin`。


2. **扫码体验**
* 在 `LIFF` 标签页找到你的 **LIFF URL**（格式为 `https://liff.line.me/你的_LIFF_ID`）。
* 将该链接发到手机上的 LINE 聊天窗口，点击即可在 LINE 内打开小程序进行调试。


3. **使用 LIFF Inspector (高级调试)**
* 可以使用 `vConsole` 或 LINE 官方提供的 `LIFF Inspector` 在 PC 端 Chrome 浏览器中调试真机的 Network 和 Console 信息。



---

## 阶段五：正式部署

1. **云端托管**
* 将前端静态文件打包，部署到支持 HTTPS 的稳定服务器或云托管平台（如 Vercel, AWS S3 + CloudFront, Firebase Hosting）。


2. **更新 Endpoint URL**
* 回到开发者控制台，将 `Endpoint URL` 修改为你正式部署的生产环境域名。



---

## 阶段六：提审与发布 (Review & Publish)

要让你的 LINE 小程序出现在 LINE 的公共搜索或服务目录中，必须通过官方审核。

### 1. 提交审核前自检 (Checklist)

| 检查项 | 说明 |
| --- | --- |
| **HTTPS 强制** | 所有资源 (API请求、图片) 必须是 HTTPS。 |
| **授权拒绝处理** | 如果用户在首次打开时**拒绝**了授权，程序不能白屏，必须有友好的提示 UI 引导用户重试或说明功能限制。 |
| **加载速度** | 首屏加载时间过长（如白屏超过 3 秒）会导致审核被拒，需加入 Loading 骨架屏或加载动画。 |
| **UI/UX 规范** | 按钮大小必须适合手指点击，不要过度依赖浏览器的返回键（建议在页面内自建返回按钮）。 |
| **法律协议** | 确保你填写的 Privacy Policy 和 Terms of Use 链接是可访问的，且内容包含获取用户数据的说明。 |

### 2. 填写审核申请

* 在 LINE Developers Console 的 Channel 设置中，点击 `Submit for review`。
* 提供用于审核的**测试账号**或**测试说明**。如果你的 App 包含支付或需要特定环境才能使用的功能，必须提供录屏视频（Demo Video）或详细的操作步骤。

### 3. 审核周期与状态

* 状态会从 `Developing` 变为 `In review`。
* 审核通常需要 **3 到 7 个工作日**。
* 如果被拒，LINE 会通过邮件发送详细的修正要求。根据邮件说明修改代码并重新部署，然后再次提交。

---

## 阶段七：上线与运营

1. **状态变更为 Published**
* 审核通过后，状态变更为 `Published`。


2. **生成推广二维码**
* 你可以生成包含 LIFF URL 的二维码，印在实体宣传物料（如店铺桌贴、名片）上。


3. **数据监控与迭代**
* 利用 LINE 官方的统计工具监控访问量。
* 后续更新代码只需推送到你的服务器端即可，**前端内容的迭代无需再次向 LINE 提交代码审核**（除非你修改了涉及隐私或核心服务条款的底层逻辑）。
